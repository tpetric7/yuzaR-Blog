If you know how to tidy up data inside of any table you are already a good data scientist! But if you want to take your skills to the next level, you should learn how to handle multiple tables inside of a data base. In this post you’ll learn how to (1) combine tables, (2) join tables to reduce duplicates, (3) unite and separate them and finally (4) how to manipulate values inside of the table in an easy way

Consider following tables:

x <- tibble(A = c("a", "b", "c"), B = c("one", "two", "three"))

y <- tibble(A = c("a", "b", "d"), B = c("one", "two", "four"),  C = c(3, 2, 1) )

z <- tibble(A = c("a", "b", "d"), B = c("one", "two", "four"),  D = c(3, 2, 1) )

x2 <- tibble(A = c("a", "c"), B = c("one", "three"))

Bind rows and bind columns

Sometimes we just need to combine several tables into one, by either combining their columns or their rows. “bind_cols()” is the most intuitive way to combine columns.

bind_cols(x, y) 
# A tibble: 3 × 5
  A...1 B...2 A...3 B...4     C
  <chr> <chr> <chr> <chr> <dbl>
1 a     one   a     one       3
2 b     two   b     two       2
3 c     three d     four      1

“bind_cols()” even renames identical column-names automatically, so that every column name is unique. But while table can have different number of columns, they need to have the same number of rows, otherwise it refuses to work and throws a following error message:

Error in `bind_cols()`:
! Can't recycle `..1` (size 3) to match `..2` (size 2)

x2
# A tibble: 2 × 2
  A     B    
  <chr> <chr>
1 a     one  
2 c     three
bind_cols(x, x2)

If you want to put two tables below each other, bind_rows() is here to help. The tables need to have the same column names thought, otherwise bind_rows() will produce NAs in your new dataset, where column names don’t match. The .id argument allows you to track to which table particular values belong.

bind_rows(y, z)
# A tibble: 6 × 4
  A     B         C     D
  <chr> <chr> <dbl> <dbl>
1 a     one       3    NA
2 b     two       2    NA
3 d     four      1    NA
4 a     one      NA     3
5 b     two      NA     2
6 d     four     NA     1
bind_rows(y, z,  .id = "table") 
# A tibble: 6 × 5
  table A     B         C     D
  <chr> <chr> <chr> <dbl> <dbl>
1 1     a     one       3    NA
2 1     b     two       2    NA
3 1     d     four      1    NA
4 2     a     one      NA     3
5 2     b     two      NA     2
6 2     d     four     NA     1
